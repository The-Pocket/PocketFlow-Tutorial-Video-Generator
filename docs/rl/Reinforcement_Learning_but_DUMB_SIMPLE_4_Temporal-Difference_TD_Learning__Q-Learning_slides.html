The TD(0) Update Rule

\[ V(S_t) \leftarrow V(S_t) + \alpha [R_{t+1} + \gamma V(S_{t+1}) - V(S_t)] \]

I know this looks complex...

but it's the entire secret.

We're going to break down every single piece of it, right now.

First, what are we replacing?

The Old Way: Monte Carlo Update

\[ V(S_t) \leftarrow V(S_t) + \alpha [\mathbf{G_t} - V(S_t)] \]

The target was G_t: the actual, final score from that point. You had to wait for it.

Temporal-Difference learning performs a brilliant, sneaky substitution.

Instead of waiting for the real outcome G_t... we create an estimate.

Monte Carlo Target

\[ G_t \]

The real future reward.
You have to wait.

TD Target

\[ R_{t+1} + \gamma V(S_{t+1}) \]

An estimated future reward.
You can calculate it immediately.

This is the "Aha!" moment.

We are replacing a REAL score...

...with an ESTIMATED score.

We're updating a guess...

...using another guess!

This is called bootstrapping, and it's a revolutionary idea.

Let's break down the TD Target:

\[ \underbrace{R_{t+1}}_{\text{Real info}} + \underbrace{\gamma V(S_{t+1})}_{\text{Our best guess}} \]

\(R_{t+1}\): The immediate reward - real, concrete information.

\(\gamma V(S_{t+1})\): The next state's value - our best guess of the future.

This whole part is the "surprise", or TD Error:

\[ V(S_t) \leftarrow V(S_t) + \alpha [\underbrace{R_{t+1} + \gamma V(S_{t+1})}_{\text{New, better estimate}} - \underbrace{V(S_t)}_{\text{Old estimate}}] \]

It’s the difference between our new estimate and our old one. The agent uses this error to nudge its old value in the right direction.

Repeat (for each episode):
  Initialize S
  Repeat (for each step):
     a. Take action A, get R, S'
     b. V(S) ← V(S) + α[R + γV(S') - V(S)]
     c. S ← S'

SARSA

What's in a Name?

It's the five pieces of information the agent uses, in order:

1. You start in a State (S)

2. You take an Action (A)

3. You get a Reward (R)

4. You land in a new State (S')

5. You choose your next Action (A')

State, Action, Reward, State, Action.

%%{init: {'theme': 'dark'}}%% graph LR S(State S) --> A(Action A) A --> R(Reward R) R --> S_prime(New State S') S_prime --> A_prime(Next Action A')

SARSA

So how do we use this? We just tweak our TD(0) formula.

Remember this?

\[ V(S) \leftarrow V(S) + \alpha[R + \gamma V(S') - V(S)] \]

Watch closely. We just swap the V's for Q's...

...and get a little more specific.

\[ Q(S,A) \leftarrow Q(S,A) + \alpha[R + \gamma Q(S',A') - Q(S,A)] \]

Look at the TD Target now:

\[ \underbrace{R + \gamma Q(S',A')}_{\text{The SARSA TD Target}} \]

We update our *current* action `Q(S,A)` based on the reward we got, plus the discounted value of the *next action A'* that we **actually decide to take**.

SARSA Algorithm (On-Policy TD Control)


Initialize Q(s,a) for all states and actions

Repeat (for each episode):
  1. Initialize state S
  2. Choose action A from S using an ε-greedy policy
  
  3. Repeat (for each step):
     a. Take action A, observe reward R and next state S'
     b. Choose the NEXT action A' from S' using the ε-greedy policy
     c. Update the Q-value for the PREVIOUS step:
        Q(S,A) ← Q(S,A) + α[R + γQ(S',A') - Q(S,A)]
     d. Update for the next loop: S ← S'; A ← A'
     
  Until S is terminal
